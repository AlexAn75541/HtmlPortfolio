1. Technical Overview

1.1 Technology Stack

Built using pure HTML5 and CSS3 without any frameworks or JavaScript dependencies. The site uses modern CSS features for responsive design and visual effects.

  • HTML5: Semantic markup for better structure and accessibility
  • CSS3: Modern styling with flexbox, CSS Grid, and custom properties
  • Google Fonts: Dosis font family for typography
  • GitHub Pages: Hosting platform with proper path configuration

1.2 Key Design Features

  • Glassmorphism Effect: Navigation bar uses backdrop-filter: blur(12px) for frosted glass appearance
  • Flexbox Layout: Vertical stacking with flex-direction: column for consistent page structure
  • CSS Variables: Centralized color scheme for easy theme management
  • Responsive Grid: Auto-fitting project cards using CSS Grid
  • Optimized Assets: Background image compressed to 25% of original size for faster loading

1.3 Technology Implementation

HTML5 Semantic Structure:

Uses semantic elements like <nav>, <header>, <main>, and <footer> for better accessibility and SEO. All pages follow a consistent structure with proper heading hierarchy.

CSS3 Flexbox:

Applied throughout the site for flexible layouts. The body uses display: flex with flex-direction: column to push the footer to the bottom. Navigation menu uses flexbox for horizontal alignment on desktop.

body {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}

CSS Grid for Projects:

Project cards use CSS Grid with auto-fit to automatically adjust columns based on screen size. Each card is minimum 280px wide and expands to fill available space.

.projects-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
    gap: 2rem;
}

Backdrop Filter Glassmorphism:

The navigation header uses backdrop-filter: blur(12px) combined with semi-transparent background to create a frosted glass effect. This modern CSS property blurs the content behind the element, creating depth and visual hierarchy.

.header-wrapper {
    background: rgba(8, 54, 110, 0.7);  /* 70% opacity blue */
    backdrop-filter: blur(12px);        /* Blur background */
    border-radius: 20px;
    width: 90%;
    max-width: 1100px;
}

Fluid Typography with clamp():

Font sizes and spacing use the clamp() function to scale smoothly between minimum and maximum values based on viewport width. This eliminates the need for multiple breakpoints while maintaining readability across all screen sizes.

.navigation-menu a {
    font-size: clamp(0.9rem, 2.2vw, 1.05rem);
    /* Min: 0.9rem, Preferred: 2.2% of viewport, Max: 1.05rem */
    padding: clamp(8px, 1.6vh, 14px) 0;
}

Google Fonts Integration:

Imported Dosis font (700 weight) via CSS @import from Google Fonts CDN for consistent typography across all browsers.

GitHub Pages Deployment:

Relative paths updated to ../content/ for proper routing on GitHub Pages. Repository configured with index.html at root for automatic deployment.

1.4 Responsive Breakpoint

Navigation adapts to screen size using a single media query at 520px: 

@media (max-width: 520px) {
    .navigation-menu {
        flex-direction: column;  /* Stack vertically on mobile */
    }
}

2. Responsive Testing

2.1 Mobile Testing (320px-480px)

Figure 1: Mobile viewport with vertical navigation

Mobile Behavior:

  • Navigation buttons stack vertically for easier touch interaction
  • Content containers expand to full width
  • Font sizes scale down using clamp() function
  • Project cards display in single column layout

2.2 Desktop Testing (1200px+)

Figure 2: Desktop viewport with horizontal navigation

Desktop Behavior:

  • Navigation buttons arranged horizontally across header
  • Content centered with max-width of 1100px
  • Project cards display in multi-column grid layout
  • Hover effects on interactive elements (buttons lift up 3px)

2.3 Browser Compatibility

Tested successfully on Chrome, Firefox, and Safari (desktop and mobile versions). The backdrop-filter property works on all modern browsers. Note: Chromium versions below 85 may have rendering issues with flex-direction: column.
